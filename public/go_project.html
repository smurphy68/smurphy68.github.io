go_project

An application created to learn basic Go, demonstrating event-driven architecture, microservices, and integration with Kafka and PostgreSQL.

Learning Outcomes

  1. Microservices in Go: Built separate broadcaster and listener services using idiomatic Go.
  2. RESTful API Design: Used Gin to expose HTTP endpoints for publishing user data.
  3. Event Streaming: Integrated Kafka for asynchronous message passing between services.
  4. Database Integration: Connected to PostgreSQL using GORM for persistent storage.
  5. Containerization: Used Docker Compose to orchestrate multi-service deployment.
  6. Error Handling: Implemented centralized error logging and handling strategies.
  7. Environment Configuration: Managed secrets and environment variables for secure deployments.

Project Architecture

  • Broadcaster Service:
    • Exposes HTTP endpoints via Gin (broadcaster/main.go).
    • Validates and publishes user data to Kafka (broadcaster/services/user_service.go).
  • Listener Service:
    • Consumes Kafka messages (listener/services/listener_service.go).
    • Persists user data to PostgreSQL (listener/main.go).
  • Shared Modules:
    • Common models (shared/models/user.go), constants (shared/consts/consts.go), and error handling (shared/shared_services/error_handling_service.go).
  • Infrastructure:
    • Kafka and PostgreSQL managed via Docker Compose (docker-compose.yml).

How It Works

  1. User data is sent to the broadcaster via HTTP POST.
  2. The broadcaster validates and publishes the data to a Kafka topic.
  3. The listener consumes messages from Kafka and saves them to PostgreSQL.
  4. All services are containerised and orchestrated with Docker Compose.
